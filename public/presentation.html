x402 Flow

1. Agent requests dataset → HTTP 402 (payment required)

2. Agent pays USDC to provider's wallet (400ms, $0.00025)

3. Agent retries with tx signature

4. Server verifies payment → returns encrypted data

5. Agent decrypts locally (AES-256)

6. Optional: EigenAI analysis with cryptographic proof

Zero human intervention. Fully autonomous.

Architecture

AI Agent (Python SDK) → HTTP 402 → Next.js API (x402 middleware)

Business Logic: Dataset | Payment | Reputation | EigenAI

Data Layer: PostgreSQL (cache) | Solana (proof) | Irys (storage)

External: Solana Devnet | Irys | EigenAI | SAS | Helius RPC

Hybrid Design: PostgreSQL (fast) + Irys (permanent) + Solana (immutable)

Reputation System

Score Calculation (0-100)

Base: 50

+ Rating: (avg - 3) × 10 (max +20)

+ Sales: min(sales/10, 20)

- Disputes: count × 30

- Refunds: count × 20

5 Badges

  • Verified (email + wallet)
  • Top Seller (100+ sales)
  • Trusted (score ≥90)
  • High Quality (rating ≥4.5)
  • Reliable (refund <5%)

On-Chain Proof

  • Irys: Permanent storage (detailed data)
  • Solana: Immutable proof (data hash)
  • SAS: Attestation (score ≥80)
  • PostgreSQL: Fast queries (<100ms)

Anti-Fraud

  • 6-layer validation
  • No rating after refund
  • One rating per order
  • Order verification required
